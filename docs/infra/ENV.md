# ENV.md — Environment Variables Reference

Complete reference for every environment variable consumed by InfluencerFlow.

Canonical source files:
- API: `api/app/config.py` (`Settings` class, pydantic-settings)
- Web: `web/.env.example` and `web/next.config.ts`
- Local defaults: `api/.env` and `web/.env` (generated by `make bootstrap`)

---

## API Service

All variables are read by the `Settings` class in `api/app/config.py`. pydantic-settings reads them in this order: actual environment → `.env` file → defaults below.

| Variable | Required | Default | Description |
|---|---|---|---|
| `ENVIRONMENT` | No | `development` | Runtime mode. `development` enables hot-reload and relaxed error output. Set to `production` on Railway. |
| `DATABASE_URL` | Yes | `postgresql+asyncpg://postgres:postgres@localhost:5432/influencerflow` | Async PostgreSQL connection string. Must include the `+asyncpg` driver. |
| `REDIS_URL` | Yes | `redis://localhost:6379/0` | Redis connection string. Used as Celery broker, result backend, and pub/sub bus. |
| `DATA_DIR` | No | `/data` | Root path for all downloaded assets, previews, edits, and exports. Must be a mounted volume in production. |
| `JWT_SECRET` | Yes (prod) | `dev-insecure-change-me` | Signs NextAuth JWTs verified by the API. Generate a random 32-byte hex string for production. |
| `FERNET_KEY` | Yes (prod) | `""` | Fernet symmetric key for encrypting OAuth refresh tokens at rest. Generate with the command below. Empty string disables encryption (dev only). |
| `SINGLE_USER_EMAIL` | Yes (prod) | `""` | Email of the single app user, seeded at first deploy. |
| `SINGLE_USER_PASSWORD` | Yes (prod) | `""` | Plaintext password for the seed user (hashed with bcrypt cost 12 on write). Only used during seeding — not stored. |
| `GOOGLE_OAUTH_CLIENT_ID` | Yes (W1+) | `""` | Google Cloud OAuth 2.0 client ID for Drive integration. |
| `GOOGLE_OAUTH_CLIENT_SECRET` | Yes (W1+) | `""` | Google Cloud OAuth 2.0 client secret. |
| `GOOGLE_OAUTH_REDIRECT_URI` | No | `http://localhost:8000/api/v1/storage/google/oauth/callback` | OAuth callback URL. Must match the URI registered in Google Cloud Console. |
| `ANTHROPIC_API_KEY` | Yes (W4+) | `""` | Anthropic API key for Claude calls. Can also be set via the `/settings` endpoint (DB value wins). |
| `REPLICATE_API_TOKEN` | Yes (W4+) | `""` | Replicate API token for LaMa, GFPGAN, and NIMA. Can also be set via the `/settings` endpoint (DB value wins). |
| `SESSION_BUDGET_USD` | No | `10.0` | Per-session soft budget cap in USD. When exceeded, the next paid operation shows an "over budget" warning and requires re-confirmation. |
| `SESSION_HARD_CAP_USD` | No | `50.0` | Per-session hard cap. Operations that would exceed this are blocked entirely regardless of confirmation. |

## Web Service

Set in `web/.env` (local) or in the Railway `web` service environment.

| Variable | Required | Default | Description |
|---|---|---|---|
| `NEXT_PUBLIC_API_BASE` | Yes | `/api/v1` | Browser-side API prefix. Used by the fetch client in the browser. Prefixed to all REST calls from client components. |
| `API_BASE` | Yes | `http://api:8000` | Server-side API host. Used by Next.js Server Components and Server Actions. Points to the internal `api` container on Railway's private network. |
| `NEXTAUTH_SECRET` | Yes (prod) | — | NextAuth session signing secret. Must be a random 32-byte string. |
| `NEXTAUTH_URL` | Yes (prod) | — | Canonical app URL, e.g. `https://influencerflow.railway.app`. Required by NextAuth in production. |

## Secrets vs Config

### Must never be committed

These contain credentials and must stay out of version control:

- `JWT_SECRET`
- `FERNET_KEY`
- `SINGLE_USER_PASSWORD`
- `GOOGLE_OAUTH_CLIENT_SECRET`
- `ANTHROPIC_API_KEY`
- `REPLICATE_API_TOKEN`
- `NEXTAUTH_SECRET`
- `DATABASE_URL` (contains password)
- `REDIS_URL` (may contain password)

`.env` files are listed in `.gitignore`. Never commit them. Use `.env.example` files (no secrets) as templates.

### Safe to commit (config, not secrets)

- `ENVIRONMENT`
- `DATA_DIR`
- `GOOGLE_OAUTH_REDIRECT_URI`
- `NEXT_PUBLIC_API_BASE`
- `NEXTAUTH_URL`
- `SESSION_BUDGET_USD`
- `SESSION_HARD_CAP_USD`

## Settings UI Override

Two AI credentials can be overridden at runtime without redeploying:

| Variable | Override endpoint |
|---|---|
| `ANTHROPIC_API_KEY` | `POST /api/v1/settings` with `{"anthropic_api_key": "..."}` |
| `REPLICATE_API_TOKEN` | `POST /api/v1/settings` with `{"replicate_api_token": "..."}` |

**DB value wins over env var.** If the database contains a value for either key, it takes precedence over the environment variable. This lets you rotate API keys from the UI without restarting the service.

To clear a DB override and fall back to the env var, set the value to an empty string via the `/settings` endpoint.

## How to Generate FERNET_KEY

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

Copy the output (a base64-encoded 32-byte key) into `FERNET_KEY` in the Railway `api` and `worker` service configs and into `api/.env` locally.

See `docs/infra/DEVOPS.md` — "Rotating Secrets" for the procedure to rotate this key after tokens have been issued.
